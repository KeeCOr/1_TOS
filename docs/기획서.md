## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 턴제 로그라이크 전투 게임 |
| 플랫폼 | Windows (Electron), 브라우저 (Next.js) |
| 핵심 루프 | 이름 짓기 → 능력치 분배 → 전투 → 보상 → 다음 층 |
| 승리 조건 | 없음 (무한 층, 최고 층수 기록) |
| 패배 조건 | HP가 0 이하가 되면 즉시 게임오버 |
| 테마 | 검사들만 사는 탑, 플레이어도 검사, 적도 검사 |

## 2. 게임 플로우

```
start
  └→ tutorial (선택)
       └→ naming          ← 캐릭터 이름 입력
            └→ stat_roll  ← 능력치 분배
                 └→ battle ─→ reward ─→ battle (반복)
                              └→ gameover (HP=0)
```

### 화면별 설명

| Phase | 설명 |
|-------|------|
| `start` | 타이틀 화면. 새 게임 / 이어하기(3슬롯) / 기록 |
| `tutorial` | 선택적 튜토리얼 (세계관 → 목표 → 전투흐름 → 시스템 8단계) |
| `naming` | 이름 입력 (최대 10자) + 8개 프리셋 이름 |
| `stat_roll` | 능력치 주사위 굴림 및 분배 |
| `battle` | 메인 전투 화면 (select_main → select_sub → rolling → result) |
| `reward` | 층 클리어 후 보상 선택 (장비 / 아이템 / 칭호 / 건너뛰기) |
| `gameover` | 사망 화면. 최고 기록 갱신 시 표시 |

## 3. 전투 시스템

### 3.1 격자 구조

```
[플레이어] ← 위치 1~5 → [적]
              거리 = 적 위치 - 플레이어 위치

행(Row): 1(위) / 2(중간, 기본) / 3(아래)
```

| 거리 | 명칭 | 색상 |
|------|------|------|
| 1 | 밀착 | 빨강 |
| 2 | 근접 | 주황 |
| 3 | 중거리 | 노랑 |
| 4 | 원거리 | 파랑 |
| 5 | 최원거리 | 보라 |

- 플레이어 시작 위치: 1, 적 시작 위치: 4 (거리 3)
- 행은 이동 서브액션(위로/아래로 이동)으로만 변경 가능
- **공격은 같은 행에만 명중**. 마법·아이템(투척)은 행 무관.

### 3.2 메인 액션 (5가지)

| 액션 | 스테미너 변화 | 특징 |
|------|-------------|------|
| 공격 | -13 | 근거리에서 강함, 원거리 약화. 같은 행에만 명중 |
| 이동 | +4 | 유일한 스테미너 회복 수단 |
| 방어 | -6 | 적 공격을 막거나 반격 |
| 마법 사용 | -16 | 원거리에서 강함, 행 무관, 쿨다운 있음 |
| 아이템 사용 | -6 | 치유 또는 원거리 투척 (행 무관) |

> **힌트 표시**: 플레이어는 적 행동 이름을 볼 수 없고 자세 힌트만 표시됨. 불가능한 이동(위치/행 경계)은 예측 목록에서 제외됨.
> **투척 아이템 vs 적 이동**: 전진하는 적에게는 항상 빗나감.

### 3.4 액션 결과 테이블

| | 공격 | 이동 | 방어 | 마법 | 아이템 |
|---|---|---|---|---|---|
| **공격** | draw/draw | win/lose | block/win | win/lose | win/lose |
| **이동** | lose/win | draw/draw | win/lose | lose/win | draw/draw |
| **방어** | win/lose | lose/win | draw/draw | lose/win | draw/draw |
| **마법** | lose/win | win/lose | win/lose | draw/draw | lose/win |
| **아이템** | lose/win | draw/draw | draw/draw | lose/win | draw/draw |

### 3.5 주사위 시스템

| 내 strength / 상대 strength | 주사위 수 |
|---------------------------|---------|
| ≥ 2.0 | 4개 |
| ≥ 1.4 | 3개 |
| ≥ 0.9 | 2개 |
| 그 외 | 1개 |

- 완벽 카운터 성공 시 +1 / 미스 시 -1 / 민첩 우위(agility ≥ 1.4)로 이동/방어 시 +1

| 모드 | 조건 | 데미지 배수 |
|------|------|-----------|
| `highest` | 기본 (최고값 사용) | 1→0.4× / 2→0.6× / 3→0.8× / 4→1.0× / 5→1.35× / 6→1.8× |
| `sum` | 모든 주사위 4 이상 (합산 크리티컬) | 1.4× ~ 2.8× |

Draw 판정: 민첩 차이 ≥ 12 → 빠른 쪽 선제(85%/25%). 그 외 힘 주사위 비교.

### 3.6 거리 보너스

| 액션 | 거리 1~2 | 거리 3 | 거리 4~5 |
|------|---------|--------|---------|
| 공격 | 1.25× | 1.00× | 0.70× |
| 마법 | 0.70× | 1.05× | 1.25× |
| 방어 (거리 1) | 1.15× | 1.00× | 1.00× |

### 3.7 스테미너 시스템

- 스테미너 < 60% → `0.65 + ratio × 0.6` (능력치 페널티)
- HP < 50% → `0.75 + ratio × 0.5` (능력치 페널티)
- 최종 능력치 = `base × min(hpFactor, staminaFactor) × conditionMultiplier`

## 4. 캐릭터 시스템

### 4.1 플레이어 생성

1. 이름 입력 (최대 10자, 프리셋: 검사/기사/방랑자/검성/암살자/수호자/전사/검귀)
2. 주사위로 능력치 결정, 재굴림 가능
3. 시작 스테미너: 22 / 마법: 1개(랜덤) / 아이템: 치유 물약 + 단검 던지기

### 4.2 컨디션 시스템

| 컨디션 | 확률 | 배수 |
|--------|------|------|
| 최상 | 10% | 1.10× |
| 좋음 | 20% | 1.05× |
| 보통 | 30% | 1.00× |
| 나쁨 | 22% | 0.85× |
| 최악 | 18% | 0.70× |

### 4.3 능력치

| 능력치 | 역할 |
|--------|------|
| strength | 공격 데미지, 주사위 개수 결정 |
| agility | 이동/방어 주사위 보너스, 민첩 선제권 판정 |
| armor | 받는 데미지 감소 (armor/100) |
| critChance | 합산 크리티컬 확률에 영향 |
| elements (fire/water/wind/earth/dark) | 마법 데미지 계산 |

### 4.4 칭호 시스템

#### 업적 칭호 (6개, 자동 획득)

| 칭호 | 획득 조건 | 보너스 |
|------|---------|--------|
| 신참자 | 처음 전투 | 없음 |
| 무기 파괴자 | 완벽 방어 10회 | strength+5, critChance+5 |
| 탑 등반가 | 5층 돌파 | agility+5 |
| 보스 사냥꾼 | 보스 처치 | strength+10, critChance+5 |
| 비전 연구자 | 10층 돌파 | critChance+5, agility+5 (마법 쿨다운 0) |
| 생존자 | 체력 25% 이하로 승리 | armor+10 |

#### 전투 칭호 (동적 생성, v1.5.4 ★)

적 처치 후 보상 화면에서 **적 능력치 기반으로 3개 생성**, 플레이어가 1개 선택.

| 생성 조건 | 칭호 예시 | 보너스 |
|---------|---------|--------|
| 적 strength 기반 | `{적 이름} 격파자` | strength += 적strength × 0.15 |
| 적 agility 기반 | `신속의 사냥꾼` / `날렵한 검사` | agility += 적agility × 0.15 |
| 적 armor 기반 | `철벽 파괴자` / `방어 격파자` | armor += 적armor × 0.25 |
| 적 최강 속성 기반 | `{속성}의 계승자` | critChance += 적critChance × 0.3 |
| 보스 처치 | `보스 처치자` | strength+8, agility+5, critChance+5 |

## 6. 진행 시스템

### 6.1 층 진행

- 적을 쓰러뜨리면 보상 화면 → 다음 층
- 층 이동 시 컨디션 재굴림
- 층수 무한 (최고 기록 localStorage 저장)

### 6.2 보상 시스템 ★ v1.5.4 전면 개편

층 클리어 후 4가지 선택:

| 선택지 | 내용 |
|--------|------|
| 장비 | 층수 기반 티어 장비 1개 |
| 아이템 | 소모 아이템 1개 (폭발 물약 / 강화 단검 / 대형 치유 물약) |
| 칭호 | 적 능력치 기반 칭호 3종 중 선택 → 즉시 능력치 반영 |
| 건너뛰기 | 아무것도 받지 않음 |
